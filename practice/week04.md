# 4주차 실습 정리



## 오늘 한 것

- 원형 / AABB / OBB Bounding Box 시각화

- 세 방식의 충돌 판정 차이 관찰



## 관찰 내용



### 원형 vs AABB

관찰 내용: 원과 사각형이 부딪힐 때, 단순히 중심점 사이의 거리를 재거나 x, y 좌표의 겹침만 확인했더니

           사각형의 모서리 부분에서 충돌 판정이 어색하게 일어남(안 부딪혔는데 부딪혔다고 인식하거나 그 반대).



### AABB vs OBB

관찰 내용: 일반적인 AABB끼리는 x와 y 좌표의 최솟값/최댓값 비교만으로 충돌을 쉽게 알 수 있었음.

           하지만 사각형에 '회전'을 적용하여 OBB 상태가 되는 순간, 사각형의 실제 차지하는 공간과 좌표값이 달라져

           기존의 단순 좌표 비교 로직이 완전히 망가짐.



## AI와의 대화에서 배운 것

배운 점: 형태가 다른 두 도형의 충돌을 정확히 판정하려면 사각형 내부에서 원의 중심과 가장 가까운 좌표(최근접점)를 먼저 찾고,

         그 점과 원 중심 간의 거리가 원의 반지름보다 작은지 비교하는 수학적 로직(Clamp 연산 등)이 필요하다는 것을 배움.

         객체가 회전하게 되면 단순히 가로/세로 범위가 아니라, 다각형의 각 면을 축으로 삼아 그림자를 투영해보는

         복잡한 수학적 접근(분리 축 정리, SAT)이 필요하다는 것을 깨달음. 상태(회전 여부)에 따라

         충돌 연산의 난이도가 기하급수적으로 올라감을 이해함.



## 내 게임에 적용한다면

OBB를 사용할 것 같다. AABB의 경우 범위가 스프라이트의 크기에 비해서 너무 크고 만약 AABB를 사용하여 게임을 만든다면

맞지 않은 것 같아보여도 맞은 것으로 컴퓨터가 인식해서 자칫하면 플레이어의 심기가 불편할 수 있을 것 같기 때문이다.
