# Week 3 실습 기록

## 목표

Pygame을 활용한 다중 객체 이동, 배경색 변경 및 달리기(대시) 기능 구현

## AI 대화 기록

**Q1: 이 코드에서 네모도 나오게 해줘**

- AI 답변: pygame.draw.rect() 함수를 사용하여 파란색 네모 객체 추가.

적용 결과: 화면에 기존 보라색 원과 함께 파란색 네모가 나타남.

배운 점: Pygame에서 사각형(rect)을 그리는 방법과 위치/크기 변수 설정법을 배움.

**Q2: 동그라미는 wasd로 움직이게 하고 네모는 방향키로 움직이게 해줘**

- AI 답변: pygame.key.get_pressed()를 활용해 원(W,A,S,D)과 네모(방향키)의 입력 키를 분리하고, 
           네모를 위한 화면 경계 제한 로직(왼쪽 위 모서리 기준) 추가.

적용 결과: 한 키보드로 두 개의 도형을 각각 조작할 수 있는 2인용 게임 형태 구현.

배운 점: 여러 객체의 키 입력을 분리해서 처리하는 방법과, 객체의 기준점에 따른 경계선 처리 방식의 차이를 배움. 

**Q3: 숫자 1, 2, 3을 누르면 일정한 색으로 변하도록 해줘**   

- AI 답변: pygame.KEYDOWN 이벤트를 사용하여 키가 '눌리는 순간'을 감지하고, 
          bg_color 변수를 업데이트하여 화면을 채우도록(screen.fill) 수정.

- 적용 결과: 1(빨강), 2(초록), 3(노랑) 키를 누를 때마다 배경색이 즉시 변경됨.

- 배운 점: 누르고 있는 상태를 확인하는 방법(get_pressed()) 외에, 키를 한 번 누르는 이벤트를 처리하는 방법(KEYDOWN)을 배움.

**Q4: shift 키를 누르면 움직이는 속도가 2배 빨라졌으면 좋겠어**  
- AI 답변: 기본 속도(base_speed)를 설정하고, pygame.K_LSHIFT 또는 K_RSHIFT 입력 여부에 따라 
          현재 속도(current_speed)를 2배로 계산하는 로직 추가.

- 적용 결과: 이동 중 Shift 키를 누르면 도형들이 2배 빠르게 이동(달리기 기능)함.

- 배운 점: 조건문을 활용해 특정 키 조합 시 객체의 속도 변수를 동적으로 변화시키는 방법을 배움.
