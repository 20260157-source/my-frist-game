# 5주차 실습 기록: 트윈 스틱 슈터 게임 개발

## 1. 선택한 템플릿 및 주제
주제: 플레이어가 적과 장애물을 피하며 생존하고 점수를 획득하는 전략적 트윈 스틱 슈팅 액션 게임

특징: WASD 이동과 방향키 사격을 분리한 조작계, 경험치 기반 궁극기 시스템, 특수 패턴을 가진 보스전 포함

## 2. 사용한 AI 프롬프트 (요약)
조작계 분리: "WASD로 이동하고 방향키로 사격이 동시에 가능한 트윈 스틱 시스템 구현 요청"

콘텐츠 구체화: "적 처치 시 경험치 드롭, 10개 수집 시 스페이스바로 화면 전체 공격(파장) 기능 추가 요청"

보스 시스템 설계: "100점마다 일반 총알이 통하지 않고 '파장'으로만 처치 가능한 보라색 보스 생성 요청"

시스템 보정: "보스 처치 시 체력 회복(최대 3개 제한) 및 비정상적인 목숨 감소 버그 수정 요청"

## 3. AI 답변에서 도움이 된 것
독립적 조작 로직: 방향 벡터(dx, dy)를 활용하여 이동 중에도 원하는 방향으로 사격이 끊기지 않게 조작계를 설계해 준 점

전략적 궁극기 설계: 단순히 적을 없애는 것을 넘어, 특정 적(보스)을 잡기 위해 궁극기를 아껴야 하는 전략적 플레이 구조를 코드화(clear())해 준 점

수치 안정화: 체력 회복 시 min(lives + 1, 3)과 같은 코드를 통해 게임의 밸런스 붕괴를 막는 예외 처리를 제공해 준 점

## 4. AI 답변을 수정하거나 버린 것
게임 오버 로직 누락: 새로운 기능(경험치/파장)을 추가하는 과정에서 AI가 실수로 게임 종료 조건을 코드에서 삭제함

현상: 목숨이 0이 되어도 게임이 끝나지 않고 -1, -2로 계속 내려가는 '불사신 버그' 발생

수정 조치: 오류가 포함된 답변의 코드를 그대로 사용하지 않고 폐기함. 이후 **"목숨 제한이 작동하지 않는다"**고 명확히 피드백하여, 게임 오버 화면과 재시작 로직이 포함된 정상 코드로 교체하여 적용함
